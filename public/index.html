Spotify CSV import format

Use this for spreadsheet lists that contain Spotify album URLs. Trackspot gets album details from Spotify and applies any optional Rating, Notes, Listen date, and Status values from the CSV.

The CSV import supports two import modes: header-based and positional.

Header-based mode

If the first non-blank row contains the header Spotify URL, Trackspot uses header-based mode. Column order doesn't matter, optional headers can be omitted, and extra columns are ignored.

Header Required Description
Spotify URL Yes A valid Spotify album URL or URI
Rating No Integer from 0–100
Notes No Free text; may contain commas, quotes, CJK characters, and newlines if quoted
Listen date No Format: YYYY-MM-DD or YYYY/MM/DD
Status No Completed, Dropped, or Planned

Note that your headers must match these exactly, including capitalization.

Positional mode

If no Spotify URL header is found, Trackspot reads columns by position. Include as many columns as you need; any trailing columns can be omitted.

Column Field Required
A Spotify URL Yes
B Rating No
C Notes No
D Listen date No
E Status No
Example 
Spotify URL,Rating,Notes,Listen date,Status
https://open.spotify.com/album/3jJ8chfB4MIsp9Z2ys43VS,97,"Notes can include commas, quotes, and Unicode like 日本語",2024-10-28,Completed
Notes
  • Spotify links may include ?si=... tracking parameters, as they will be stripped automatically.
  • Rows without a valid Spotify album link are skipped.
  • Invalid rating, date, or status values don't block a row from importing; Trackspot imports what it can and logs a warning.
  • Missing optional fields default to blank, except Status, which falls back to the default you choose in the import UI.
  • Album names, artist names, art, and release details come from Spotify, so they are not required in the import CSV.
  • If the same album appears more than once, only the first row is imported.
